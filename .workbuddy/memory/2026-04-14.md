# 2026-04-14 工作日志

## qmlls 读取 qmltypes 报错修复

**错误：** `读取typeinfo文件发生错误：File "E:/qt/Mosui/shared/MosuiBasic/MosuiBasic.qmltypes" does not exist`

**根因：** `MosuiBasic/.qmlls.ini` 和 `Mosuiapp/.qmlls.ini` 的 `importPaths` 只包含 Qt 自带 qml 路径，缺少 `E:/qt/Mosui/shared`，导致 qmlls 无法在正确位置找到 MosuiBasic 模块的 qmltypes 文件。

**修复：** 两个 `.qmlls.ini` 的 `importPaths` 追加 `E:/qt/Mosui/shared`，并在 VSCode 中 Reload Window。


**问题：** `iconSource: 0xF159` 传入后图标仍然空白

**根因：** `iconDelegate`（Component）内 `visible: !isIcon` 中的 `isIcon` 是 `MosIconText` 自身的属性，依赖 `iconSource` 绑定同步。QML Loader 加载 Component 时存在时序竞争：先求值 `isIcon`（此时 `iconSource` 还是默认 `0`，`isIcon=true`），再同步 `root.iconSource`，导致第一帧 `visible=false` 且不再更新。

**修复：** 将 `visible: !isIcon` 改为直接引用 `root` 属性：
`visible: root.loading || (root.iconSource !== 0 && root.iconSource !== "")`

## MosIconButton 图标不显示修复

**问题：** `MosIconButton` 的 icon 没有正常加载（显示方块或空白）

**根因：** 字体 family name 写错。`MOSUI-Icons.otf` 文件内实际注册的 family name 是 **`MOSUI-Font`**（通过读取 name table 确认），但两个 QML 文件均写错：
- `MosIconText.qml`：写的是 `'MosUI-Icons'`
- `MosIconButton.qml`：写的是 `'MOSUI-Icons'`

**修复：** 两个文件的 `font.family` 统一改为 `'MOSUI-Font'`。

**教训：** 图标字体 family name 必须与 otf/ttf 文件内嵌 name table 一致，文件名≠family name。
可用 `QFontDatabase::applicationFontFamilies()` 运行时验证。
